## Bugs Found This Session

### Bug #1 — PWA Banner Reappears After Dismissal (P1) → FIXED
**Symptom:** Clicking "Not now" on install banner hid it, but navigating to another page brought it back.
**Root cause:** `dismissed` state was `useState(false)` — resets on every Next.js page navigation.
**Fix:** Added `sessionStorage` dual-layer: checks `pwa-banner-dismissed-session` on mount; sets it on dismiss. SSR guard added.
**File:** `packages/frontend/components/InstallPrompt.tsx`
**Push status:** PENDING — included in Patrick's push block.

### Bug #2 — Shopper Onboarding Popup Fires on Wrong Pages (P2) → FIXED
**Symptom:** Shopper welcome popup appearing on /inspiration, /shopper/favorites, /shopper/messages, and any page that isn't the homepage.
**Root cause:** Shopper `OnboardingShower` in `_app.tsx` had no route guard (unlike the organizer variant which already had `orgPages`).
**Fix:** Added `shopperFirstPages = ['/', '/trending', '/index']` allowlist — popup only fires on first-visit pages.
**File:** `packages/frontend/pages/_app.tsx`
**Push status:** PENDING — included in Patrick's push block.

### Bug #3 — Inspiration Page: All Item Images Broken (P2) → FIXED
**Symptom:** Every item card on /inspiration showed a broken image icon.
**Root cause:** Seed data items have no Cloudinary photos (`photoUrls` is empty array). No fallback handling in `InspirationGrid.tsx`.
**Fix:** Added `onError` handler on `<Image>` tag to hide broken images, plus a styled gray placeholder div with "No image" label shown when `photoUrls?.[0]` is absent.
**File:** `packages/frontend/components/InspirationGrid.tsx`
**Push status:** PENDING — included in Patrick's push block.
