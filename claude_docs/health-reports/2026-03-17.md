Overall codebase health is strong. No secrets leaked, no SSR crashes, no production console.log statements, no broad Prisma includes, and all admin/auth routes are properly gated. ## High (fix this week)

- **MAILERLITE_API_KEY vs MAILERLITE_API_TOKEN name mismatch** — `packages/backend/.env` has `MAILERLITE_API_TOKEN` but `mailerliteService.ts:15` and `snoozeService.ts:16` read `process.env.MAILERLITE_API_KEY`. The service gracefully skips (warns to console) when the key is absent, so no crash — but MailerLite subscriber sync and snooze functionality is silently disabled in production. Fix: rename `MAILERLITE_API_TOKEN` to `MAILERLITE_API_KEY` in the Railway env vars (Patrick action). Also update `.env.example` to remove the `MAILERLITE_API_KEY` entry and add `MAILERLITE_API_TOKEN` — or standardize to `MAILERLITE_API_KEY` everywhere.

---

## Medium (fix this sprint)

- **Unauthenticated photo share/like POST routes** — `photoOps.ts:29-31`: `POST /:stationId/shares` and `POST /shares/:shareId/like` have no `authenticate` middleware and no rate limiter. Any unauthenticated client can flood photo shares or spam likes. The write endpoints for station management are correctly protected, making this an inconsistency. Fix: add `authenticate` (optional auth is acceptable to allow guest sharing, but add a rate limiter regardless) or document as intentional guest feature.

## Low (track, fix when relevant)

- **8 DEFAULT_* env vars in .env.example not set in .env** — `DEFAULT_CITY`, `DEFAULT_COUNTY`, `DEFAULT_LAT`, `DEFAULT_LNG`, `DEFAULT_RADIUS_MILES`, `DEFAULT_STATE`, `DEFAULT_STATE_ABBREV`, `DEFAULT_TIMEZONE` are all in `.env.example` but absent from the Railway `.env`. All have safe hardcoded fallbacks in `regionConfig.ts` (e.g., lat/lng default to Grand Rapids center), so no runtime error. Risk: if Railway env is ever reset, GR defaults kick in silently. Fix: add these to Railway env vars for explicitness (Patrick action, low urgency).

- **NEXT_PUBLIC_STRIPE_TERMINAL_SIMULATED undocumented** — `.env` has this var but `.env.example` doesn't. Should be added to `.env.example` with a comment so future deploys document it.

- **POST /routes/plan has no auth** — `routes.ts:7` registers a public route to an AI route planner. If this involves any LLM call (Anthropic/OpenAI), it's an unbounded compute endpoint. Confirm whether rate limiting exists in the controller; if not, add it.
