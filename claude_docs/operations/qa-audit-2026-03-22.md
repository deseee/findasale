## VERDICT: ⛔ NO-GO

Two P0 blockers prevent a clean beta launch: the messages thread page renders blank for all users, and the Stripe checkout is completely broken (no user can upgrade). Fix those two before any public-facing beta recruitment.

---

## P0 — CRITICAL (must fix before any beta)

### BUG-01 · `/messages/[conversationId]` blank page for all roles
**Pages:** `/messages/[id]`, `/organizer/messages` threads
**Symptom:** Thread page renders completely blank. DOM contains full content (confirmed via `body.innerText` and computed styles) but the `main` element has zero BoundingClientRect dimensions (`{}`). `msgContainerHeight: 0`.
**Root cause:** `min-h-screen flex flex-col` on the thread component collapses inside the Layout's flex wrapper — the child needs `flex: 1` not `min-h-screen` to fill available height.
**Impact:** 100% of message threads are inaccessible to shoppers and organizers. Core communication feature dead.

### BUG-02 · Stripe checkout broken — `POST /api/billing/checkout` → 404
**Page:** `/organizer/upgrade` ("Start Free Trial" button)
**Symptom:** Clicking "Start Free Trial" fires `POST https://finda.sale/api/billing/checkout` which returns 404 HTML. Frontend parses it as JSON → `SyntaxError: Unexpected token '<', "<!DOCTYPE "... is not valid JSON`. No Stripe redirect occurs.
**Root cause:** The Next.js API route at `/api/billing/checkout` is missing or was never deployed.
**Impact:** Zero users can upgrade from Free to PRO. All revenue conversion is blocked.

## HIGH — Fix before beta opens to organizers

### BUG-03 · "Manage Plan" dead-end — no Stripe Customer Portal
**Page:** `/organizer/premium` → "Manage Plan" → `/organizer/subscription`
**Symptom:** PRO users clicking "Manage Plan" land on a page that reads "Subscription managed externally or not yet configured." No link to Stripe Customer Portal. PRO users cannot cancel, update payment, or view invoices without emailing support.
**Impact:** All existing paid users are locked out of subscription self-service.

### BUG-04 · `/admin/invites` crashes for ADMIN user
**Page:** `/admin/invites`
**Symptom:** "Something went wrong" error boundary. Console: `TypeError: n.map is not a function` in `admin/invites-087639de1ba9b419.js:1:5547`.
**Root cause:** Component calls `.map()` directly on the API response object instead of destructuring the invites array property. The endpoint returns `{ invites: [...] }` but code does `response.map(...)`.
**Impact:** Admin cannot access beta invite management at all.

### BUG-05 · Follow button fires zero network requests
**Page:** `/sales/[id]` — "+ Follow" button
**Symptom:** Clicking "+ Follow" produces no network request, no state change, no feedback. Button stays as "+ Follow".
**Also:** `GET /api/organizers/[id]/follow-status` → 404 on page load (endpoint not registered).
**Impact:** Organizer follower system entirely non-functional.

## MEDIUM — Fix before public launch

### BUG-06 · Broken images on Trending and Inspiration Gallery
**Pages:** `/trending`, `/inspiration`
**Symptom:** 8/9 images broken on Trending, all item images blank on Inspiration. Source URLs use `fastly.picsum.photos` domain.
**Root cause:** `fastly.picsum.photos` not whitelisted in `next.config.js` images domains array.
**Fix:** Add `fastly.picsum.photos` to `next.config.js` `images.domains`.

### BUG-07 · Edit Sale — dates not pre-populated
**Page:** `/organizer/edit-sale/[id]`
**Symptom:** Start Date and End Date fields show `mm/dd/yyyy` placeholder even when the sale has existing date values. Organizer could save sale without dates.
**Root cause:** Date inputs not pre-populated from fetched sale data.

### BUG-08 · Edit Item — Category not pre-populated
**Page:** `/organizer/edit-item/[id]`
**Symptom:** Category dropdown shows "Select a category" despite item having `category: "vintage"` in the database.
**Root cause:** Category select not seeded with the current item value on mount.

### BUG-09 · `GET /api/organizers/[id]/follow-status` → 404
**Triggered:** On every sale detail page load (2 calls per load).
**Impact:** Console noise on every sale page. Contributes to BUG-05.

### BUG-10 · `GET /api/sales/[id]/photo-ops` → 404
**Triggered:** On sale detail page load.
**Impact:** Photo ops feature silently broken.

### BUG-11 · Sale detail over-fetches same endpoint 8+ times
**Page:** `/sales/[id]`
**Symptom:** `GET /api/sales/[id]` called 8+ times on a single page load — multiple hooks/components independently fetching without a shared cache.
**Impact:** Unnecessary backend load, slower page response.

### BUG-12 · Favorites N+1 query — 14 individual item calls
**Page:** `/shopper/favorites`
**Symptom:** 14 individual `GET /api/favorites/item/[id]` calls, one per item.
**Fix:** Batch endpoint `GET /api/favorites/items?ids=...`.

### BUG-13 · `POST /api/sales/.../ripples` → 403 on shopper page load
**Symptom:** Fires automatically on sale detail load for non-authorized roles, returns 403. Should not fire or should be silently swallowed for shoppers.

### BUG-14 · Pricing page shows organizer-centric features to shoppers
**Page:** `/pricing`
**Symptom:** A logged-in shopper sees "Your Plan" with organizer metrics (items/sale, photos/item, AI tags). No role differentiation for shopper context.

### BUG-15 · Billing section hardcoded light theme — broken in dark mode (systemic)
**Pages:** `/organizer/premium`, `/organizer/upgrade`
**Symptom (multiple):**
- PRO pricing card: light mint background with near-invisible white feature text
- Compare All Plans table: white row backgrounds with black text inside dark theme
- Upgrade page header: "Choose Your Plan" heading nearly invisible against mint background
- Free plan card on upgrade page: same light background, faint text
**Root cause:** Entire billing section uses hardcoded light-mode classes (`bg-green-50`, light card variants) with no `dark:` overrides.
**Note from Patrick:** Also observed "Today's Treasure Hunt" card on homepage with cream background — same pattern may exist outside billing section.

### BUG-16 · Dual stacked status badges on items (AVAILABLE + PENDING REVIEW)
**Page:** `/organizer/add-items/[id]`
**Symptom:** Each item shows both a green "AVAILABLE" badge and an amber "PENDING REVIEW" badge stacked. Single `status: "AVAILABLE"` in database. "PENDING REVIEW" appears to be driven by `isAiTagged: false` but renders as a confusing second status with no explanation.

## LOW — Polish before public launch

### BUG-17 · Raw unicode `\u25cf` in LIVE status badge (Edit Sale)
**Page:** `/organizer/edit-sale/[id]`
**Symptom:** Status badge renders literal text `\u25cf LIVE` instead of `● LIVE`.

### BUG-18 · Raw unicode `\u2014` in Neighborhood label (Edit Sale)
**Page:** `/organizer/edit-sale/[id]`
**Symptom:** Label reads `Neighborhood (optional \u2014 helps shoppers find you)` instead of em dash `—`.
**Pattern:** Both BUG-17 and BUG-18 are JSX string literals with unicode escapes — fix by using actual characters or HTML entities.

### BUG-19 · `aiConfidence: 0.5` displayed as "50%" badge on Inspiration items
**Page:** `/inspiration`
**Symptom:** Internal AI confidence score rendered as an orange percentage badge — looks like a discount to shoppers.

### BUG-20 · Sentry error reporting returning 503
**Symptom:** All Sentry envelope POST requests return 503. External error monitoring completely non-functional.

### BUG-21 · Bare "0" renders on sale detail below organizer badge
**Page:** `/sales/[id]`
**Symptom:** An unlabeled "0" renders below the "New Organizer (0 sales so far)" badge — likely an unguarded value render.

### BUG-22 · Wrong redirect for authenticated wrong-role user accessing organizer routes
**Symptom:** Shopper accessing `/organizer/dashboard` redirected to `/login` instead of a 403/access-denied page. User is already logged in.

### BUG-23 · Admin auth redirects to `/` instead of `/login?redirect=...`
**Symptom:** Both unauthenticated and non-admin users hitting `/admin` are redirected to homepage. Should redirect to `/login?redirect=/admin` for better UX. Security is intact — no data exposure.

### BUG-24 · PWA install prompt appears on every page, no suppression
**Symptom:** "Add FindA.Sale to your home screen" banner fires on every page navigation, including immediately after dismissing on the previous page.

### BUG-25 · Dead route aliases
**Symptom:** `/create-sale`, `/manage-sales`, `/organizer/manage-sales` all return 404. Correct routes are `/organizer/create-sale` and `/organizer/sales`.
