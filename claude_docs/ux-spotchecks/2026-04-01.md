### Issues Found

**pages/index.tsx**

- Search API has no error state ‚Äî `useQuery(['search', searchQuery])` handles `isSearching` but never checks `isError`. If the search endpoint fails, the skeleton disappears and the user sees a blank grid with no message ‚Äî `pages/index.tsx` (lines 125‚Äì532) ‚Äî **severity: medium**

- "üíæ Save This Search" button is visible to unauthenticated users. Clicking it triggers an API call that returns a 401; the catch block shows the generic "Failed to save search. Please try again." toast. A logged-out user has no way to know they need to sign in first ‚Äî `pages/index.tsx` (lines 335‚Äì343) ‚Äî **severity: medium**

- Hero search `<input>` has no `<label>` or `aria-label`. The search icon is visual only. Screen readers announce this field without context ‚Äî `pages/index.tsx` (line 327) ‚Äî **severity: medium**

- Four `console.error` / `console.warn` calls present in production code (geolocation timeout, geolocation error, feed fetch error, save-search error) ‚Äî `pages/index.tsx` (lines 115, 139, 148, 248) ‚Äî **severity: low**

**pages/sales/[id].tsx**

- Bid input (`<input type="number">`) has no associated `<label>` or `aria-label`. The `placeholder="Enter bid amount"` text is not a label substitute ‚Äî `pages/sales/[id].tsx` (line 956) ‚Äî **severity: medium**

- `window.confirm()` used for the organizer "End Auction" action. Native browser dialogs block the UI thread, are unstyled, and are suppressed in some PWA/embedded WebView contexts. Any user-facing confirmation should be a modal ‚Äî `pages/sales/[id].tsx` (line 1020) ‚Äî **severity: medium**

- "Place Bid" button (lines 1049‚Äì1056) renders when `sale.isAuctionSale && item.auctionEndTime` with no check that the auction end time is in the future. An auction whose timer has lapsed but whose status hasn't been updated to `AUCTION_ENDED` will show a clickable "Place Bid" button that opens the bid input, while the bid input itself (line 953) correctly checks `new Date(item.auctionEndTime) > new Date()` and won't render ‚Äî leaving the user with a button that appears to do nothing ‚Äî `pages/sales/[id].tsx` (lines 1049‚Äì1056) ‚Äî **severity: low**

### Clean

- Loading states: both pages use Skeleton components correctly during data fetches. Sale detail page has a full-skeleton screen. Homepage has per-card skeletons.
- Error states: feed error (`isError`) handled with a retry button. Sale 404 and 429 handled with distinct messages.
- Empty states: homepage covers 3 scenarios (no results, no sales in area, filters produce nothing). Sale detail covers the no-items case with a CTA for the organizer.
- SSR safety: `navigator.geolocation` and `localStorage` access are inside `useEffect`. All `window.*` calls in event handlers or guarded with `typeof window !== 'undefined'`. No bare SSR-unsafe globals.
- No hardcoded placeholder text ("Coming soon", "TODO") found in rendered UI.
- Sale detail main gallery, thumbnails, and item images all have meaningful `alt` text.
- Share buttons and "Take a Tour" button have proper labels/aria-labels.
- Dark mode classes applied consistently throughout both files.

### Recommended Fixes

1. **(medium) Add search error state in index.tsx** ‚Äî Add `isSearchError` check after `isSearching` in the search results section and render an `EmptyState` or brief error message when the search API fails.

2. **(medium) Gate "Save This Search" behind auth** ‚Äî Either hide the button for unauthenticated users (show "Sign in to save searches" link instead), or update the error toast to "Sign in to save searches" when a 401 is returned.

3. **(medium) Add aria-label to homepage search input** ‚Äî `aria-label="Search sales by title, city, or keyword"` on the hero `<input>`.

4. **(medium) Add label to bid input in sales/[id].tsx** ‚Äî Wrap with `<label>` or add `aria-label="Bid amount"` to the bid `<input type="number">`.

5. **(medium) Replace window.confirm with a modal for End Auction** ‚Äî Use the existing modal pattern (e.g. a small inline confirmation card or a lightweight Dialog component) instead of the native browser dialog.

6. **(low) Fix "Place Bid" button guard for expired auctions** ‚Äî Add `&& new Date(item.auctionEndTime) > new Date()` to the render condition at line 1049 to match the bid input's own guard at line 953.

7. **(low) Remove console.error/warn from index.tsx production paths** ‚Äî Replace with no-ops or conditional dev-only logging.
