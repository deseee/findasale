- `*-audit-*.md` — Structured reviews of systems (beta readiness, code quality, UX flows, etc.)
- `payment-stress-test.md` — Stripe payment path testing results
- `ux-*.md` — User experience audits and verification reports

- `2026-03-*.md` — Daily or periodic session wrap files
- `BACKLOG_*.md` — Backlog snapshot files

- `*-fleet-*.md` — Agent/subagent fleet evaluations

- `2026-03-*.md` — Dated health check reports
- `*-health-*.md` — Specific system health reports
- `*-dev-handoff-*.md` — Handoff reports from health checks

- `*-onboarding-*.md` — Organizer and shopper onboarding materials
- `success-*.md` — Success criteria and tracking

Marketing strategy, content calendars, social media content, and marketing-related research.
- `content-*.md` — Content planning and drafts

### `qa/`
QA reports, test results, and payment edge case documentation.
- `payment-edge-cases-*.md` — Payment system edge cases and fixes
