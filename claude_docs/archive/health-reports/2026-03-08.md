# Health Report — 2026-03-08

## Summary

Overall codebase health is **Green**. No critical security issues found. All mutating routes are properly authenticated (most via `router.use(authenticate)` at the router level). No hardcoded secrets in production code. The top concerns are: (1) `.env.example` drift with 8 missing default location vars and a renamed MailerLite key, (2) many `findMany()` calls without explicit `take` limits in job/service files, and (3) minor frontend console.log/warn statements in production code paths.

## Critical (blocker — fix before next deploy)

None

## High (fix this week)

### H1 — .env.example drift: missing DEFAULT_* location vars
`packages/backend/.env.example` is missing 8 keys present in `.env`:
- `DEFAULT_CITY`, `DEFAULT_STATE`, `DEFAULT_STATE_ABBREV`, `DEFAULT_LAT`, `DEFAULT_LNG`, `DEFAULT_RADIUS_MILES`, `DEFAULT_COUNTY`, `DEFAULT_TIMEZONE`

These are needed for the search/geo defaults. New developers cloning the repo won't have them.

**File:** `.env.example` (root) and `packages/backend/.env.example`

### H2 — .env.example key mismatch: MAILERLITE_API_KEY vs MAILERLITE_API_TOKEN
`.env.example` declares `MAILERLITE_API_KEY` but `.env` uses `MAILERLITE_API_TOKEN`. If the code references one name, the example file should match.

**File:** `packages/backend/.env.example`

### H3 — Root .env.example missing VAPID_* push notification keys
Root `.env.example` is missing `VAPID_PUBLIC_KEY`, `VAPID_PRIVATE_KEY`, `VAPID_CONTACT_EMAIL` which are present in `.env`. Push notifications won't work for new devs without these.

**File:** `.env.example`

## Medium (fix this sprint)

### M1 — findMany() without take limits (30+ occurrences)
Many `findMany()` calls in controllers and services lack a `take` limit. While most are scoped by `where` clauses, some could return unbounded result sets on large datasets:

Key files with multiple unbounded findMany calls:
- `controllers/userController.ts` (6 calls)
- `controllers/adminController.ts` (5 calls)
- `services/weeklyEmailService.ts` (4 calls)
- `services/wishlistMatchEmailService.ts` (3 calls)
- `controllers/disputeController.ts` (3 calls)
- `controllers/favoriteController.ts` (2 calls)
- `controllers/itemController.ts` (3 calls)

**Risk:** Performance degradation as data grows. Not urgent at beta scale but should be addressed before public launch.

### M2 — Frontend console.warn in production code paths
A few `console.warn` calls in frontend production code:
- `hooks/usePushSubscription.ts:62` — "Push subscription failed"
- `pages/map.tsx:66` — "Geolocation denied or unavailable"

These are non-sensitive warning messages (no secrets leaked), but should be behind a debug flag or removed for production polish.
