## High (fix this week)

### 1. SSR-unsafe window/localStorage access without checks
- **File:** `/sessions/lucid-dazzling-hawking/mnt/FindaSale/packages/frontend/pages/register.tsx:28`
- **Issue:** `new URLSearchParams(window.location.search)` called in useEffect but `window` accessed at top level in component before useEffect guard. SSR renders this on server where `window` is undefined.
- **Impact:** Potential hydration mismatch or runtime error during initial load.
- **Fix:** Wrap in `if (typeof window !== 'undefined')` or move inside useEffect.

### 2. SSR-unsafe localStorage access without guard
- **File:** `/sessions/lucid-dazzling-hawking/mnt/FindaSale/packages/frontend/pages/refer/[code].tsx:16`
- **Issue:** `localStorage.setItem()` called directly without checking if `window` is defined. Will throw on server-side render.
- **Impact:** SSR failure, hydration mismatch.
- **Fix:** Check `typeof window !== 'undefined'` before accessing localStorage.

### 3. SSR-unsafe window.location/document access
- **File:** `/sessions/lucid-dazzling-hawking/mnt/FindaSale/packages/frontend/pages/sales/[id].tsx:141,220,223,226`
- **Issue:** Multiple window/document APIs (`window.location.search`, `document.createElement()`, `document.body.appendChild()`) called without SSR safety guards.
- **Impact:** Server-side render will fail or throw errors.
- **Fix:** Wrap all browser API calls with `typeof window !== 'undefined'` or move to useEffect.

### 4. Unpaginated Prisma findMany calls (performance risk)
- **File:** `/sessions/lucid-dazzling-hawking/mnt/FindaSale/packages/backend/src/controllers/affiliateController.ts:63`
- **Issue:** `findMany()` without `take:` limit. Could return unbounded result set.
- **Impact:** High memory usage, slow response times if dataset grows (N+1 queries, large serialization).
- **File:** `/sessions/lucid-dazzling-hawking/mnt/FindaSale/packages/backend/src/controllers/bountyController.ts:68,90`
- **File:** `/sessions/lucid-dazzling-hawking/mnt/FindaSale/packages/backend/src/controllers/favoriteController.ts:55,89`

- **And 20+ other findMany calls across controllers**
- **Fix:** Add `take: N` and `skip: offset` pagination to all findMany calls. Use cursor-based pagination for large datasets.
