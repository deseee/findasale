## High (fix this week)

- **Password reset token logged to stdout** — `packages/backend/src/routes/auth.ts:96` — When Resend is not configured (the `else` branch), the full reset URL including the token is logged via `console.log`. In production with Resend active this branch doesn't fire, but if Resend ever fails or the env var is missing, tokens leak to server logs. Replace with a redacted log or remove entirely.

While the actual values are not logged, "✅ Present" / "❌ Missing" patterns in logs can aid reconnaissance. Consider removing or gating behind `NODE_ENV=development`.
